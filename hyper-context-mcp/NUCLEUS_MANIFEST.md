# THE NUCLEUS MANIFEST
**The Immutable Laws of the Sovereign Agentic Operating Layer**

The `hyper-context-mcp` directory is the **Stem Cell** (The Nucleus) of the entire ecosystem. To ensure this system retains infinite potential and can spawn any type of software or operational node, the following architectural laws are immutable.

## 1. Absolute Encapsulation
The Nucleus shall never contain localized business logic. It must not house consumer applications, API endpoints for specific SaaS products, or hardcoded scripts that are highly specific to a single domain.

## 2. The Orchestration Mandate
The Nucleus has only three responsibilities:
- **Thinking:** Maintaining the persistent memory graph (`network-state.json`).
- **Monitoring:** Sensing the environment through ambient file listeners and live infrastructure bridges.
- **Executing:** Dispatching abstract, self-healing state machines.

## 3. Cellular Spawning
When the engine is commanded to execute a protocol (e.g., building an app), it **must branch out**. However, the Nucleus will not magically create directories for you. It relies entirely on the Protocol Schema to explicitly construct an isolated "Child Cell" outside of the Nucleus root (e.g., via `mkdir` or `git clone`). The engine then safely executes all compiling, testing, and deployment commands inside that external cell.

## 4. Symbiosis over Monolith
By keeping the Nucleus lightweight, we enable true symbiosis. The Nucleus can spawn a SaaS Launchpad cell in directory A, a DevSecOps auditor cell in directory B, and orchestrate them to communicate and verify one another. A monolithic design would shatter this capability.
